# 刷记 Release Notes

## v1.5.4 (2026-06-18)

### 修复

- 修复自定义主题色取色器：再次打开时停留在当前已选颜色，而不是每次回到默认色。
- 修复主页「未分类」分组标题在英文界面下仍显示中文的问题。
- 修复卡片详情页「刷卡记录」的笔数被显示成「N / N」的文案错误，现正确显示为刷卡笔数。

### 优化

- 主页整体进度统计改用缓存计算，卡片较多时滚动更流畅。
- 启动续期逻辑收口到应用容器内部，移除一处脆弱的向下类型转换；清理冗余 DAO 命名、未使用的字符串资源与无效的文本溢出配置。
- 内部代码质量与一致性改进。

## v1.5.3 (2026-06-18)

### 新增

- **应用内语言切换**：在「设置 → 语言」可在「跟随系统 / 简体中文 / English」之间切换，切换后即时生效，并由系统记忆所选语言。
- 新增完整英文界面（English），全部界面文案均已本地化。

### 优化

- 外观设置各项补全图标，视觉更统一。
- 全量界面文案（含导入 / 导出的错误提示）抽取为字符串资源，为后续多语言扩展打好基础。
- 内部代码质量改进。

## v1.5.2 (2026-06-17)

### 修复

- **自定义主题色现在会作用于整个界面**。此前选择自定义颜色时只有按钮和强调文字会变色，背景与卡面仍停留在默认的蓝绿色调；现已修复，背景、各级表面与容器颜色都会随所选颜色一同协调变化。

### 优化

- 自定义配色改用 Material 3 官方动态取色方案生成整套颜色，观感与系统动态色保持一致、过渡更自然。
- 内部稳定性与代码质量改进。

## v1.5.1 (2026-06-17)

- 卡组织图标替换为官方规范 SVG（Visa / MasterCard / JCB / UnionPay / American Express / Diners Club / Discover）
- 修复卡组织图标在卡片上的渲染变形与比例问题
- 修复 Android SDK Color 函数不可用导致的编译错误
- 主题色 HSL 计算改用纯 Kotlin 实现，解除对 `android.graphics.Color` 的依赖
- `SettingsScreen` 补充缺失 import，保证 release 构建链路完整

## v1.5.0

- 新增信用卡年费结算日提醒
- 新增卡片有效期记录
- 支持横版 / 竖版两种卡面布局
- 引入 skydoves/colorpicker-compose 现代化调色板（HSV 色环 + 亮度/透明度/饱和度滑块）
- 数据库升级至 v2，新增 `image_source_type` / `image_provider_key` / `card_orientation` 三列

## v1.4.0
